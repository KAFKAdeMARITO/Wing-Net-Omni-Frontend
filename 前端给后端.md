# 前端给后端 —— 翼网全境 (Wing-Net Omni) 前端实现清单与对接指南

> **文档定位**：本文档由前端开发者撰写，面向后端 NS-3 C++ 引擎开发者，详细说明前端已实现的全部功能模块、数据消费方式、以及联调时后端需要满足的数据输出规范。

---

## 一、 技术架构概览

### 1.1 技术栈
| 层级 | 技术选型 |
|------|---------|
| 框架 | Vue 3 (Composition API) + Vite |
| 3D 渲染 | Three.js + EffectComposer (Bloom 后处理) |
| 图表 | Apache ECharts 5 |
| 动画 | GSAP (GreenSock) |
| CSV 解析 | PapaParse |
| 语言 | TypeScript |

### 1.2 三段式架构
前端遵循 **"沙盘布景 → 底层引擎精算 → 前端高频无损回放"** 的三段式架构：

```
┌─────────────────┐     ┌──────────────┐     ┌─────────────────┐
│  场景编辑器      │     │ NS-3 C++ 引擎 │     │ 前端回放引擎     │
│  (ScenarioEditor)│────▶│  (后端精算)    │────▶│ (PlaybackEngine) │
│  输出 scene.json │     │ 输出 4 个 CSV  │     │ 读取 CSV → 帧动画 │
└─────────────────┘     └──────────────┘     └─────────────────┘
```

### 1.3 数据流模式
前端支持 **两种数据源**，通过 URL 参数 `?mode=csv` 切换：

| 模式 | URL | 数据来源 | 用途 |
|------|-----|---------|------|
| Mock | `localhost:5174` (默认) | 前端内置 `mockData.ts`，300 帧仿真数据 | 独立开发/演示 |
| CSV  | `localhost:5174?mode=csv` | 读取 `/public/data/` 下 4 个 CSV 文件 | 联调/正式路演 |

> CSV 模式下可通过 `?mode=csv&path=/custom/path` 指定自定义数据目录。

## 二、 前端消费的 CSV 文件清单

前端通过 `src/services/csvParser.ts` 并行读取以下 **4 个 CSV 文件**，合并为统一的帧数据流：

### 2.1 `qos_performance.csv` — 核心网络质量表
```csv
time,avg_pdr,avg_throughput,avg_delay,uav0_pdr,uav0_delay,uav0_throughput,...uav14_pdr,uav14_delay,uav14_throughput
0,0.95,120.5,12.3,0.97,10.2,125.0,...
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `time` | int | 仿真时间戳（秒），作为帧 tick 索引 |
| `avg_pdr` | float | 全网平均投递率 (0~1) |
| `avg_throughput` | float | 全网平均吞吐量 (Mbps) |
| `avg_delay` | float | 全网平均时延 (ms) |
| `uavX_pdr` | float | 第 X 架无人机 PDR |
| `uavX_delay` | float | 第 X 架无人机时延 (ms) |
| `uavX_throughput` | float | 第 X 架无人机吞吐量 (Mbps) |

**前端消费方式**：
- `avg_pdr × 100` → 左侧面板"平均 PDR"百分比 + QoS 健康圆环分数
- `avg_delay × 1.5` → 左侧面板"P99 时延"（估算值）
- `avg_throughput` → 左侧面板"总吞吐" + 右侧折线图
- `uavX_pdr/delay/throughput` → 点击无人机后弹窗中的详细网络指标

### 2.2 `resource_allocation.csv` — 信道功率分配表
```csv
time,uav0_ch,uav0_pwr,uav0_rate,uav1_ch,uav1_pwr,uav1_rate,...
0,0,20.0,125.0,1,18.5,122.3,...
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `uavX_ch` | int | 信道号 (0/1/2 → 对应 CH1/CH2/CH3) |
| `uavX_pwr` | float | 发射功率 (dBm) |
| `uavX_rate` | float | 传输速率 |

**前端消费方式**：
- `_ch` → 3D 沙盘中无人机发光颜色（CH1=青, CH2=紫, CH3=绿）
- `_ch` → 右侧频谱柱状图（统计各信道使用数）
- `_ch` → 左侧活动节点档案的信道标签
- `_pwr` → 左侧功率光环长度 + 右侧"均方功率"指标
- 同频+近距离 → 前端自动判定为"同频冲突"，触发红色告警特效

### 2.3 `topology_evolution.csv` — 全局拓扑连通表
```csv
time,num_links,connectivity
0,28,0.97
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `num_links` | int | 当前时刻有效微波链路总数 |
| `connectivity` | float | 全局连通率 (0~1) |

**前端消费方式**：
- `num_links` → 左侧面板"活跃链路"数字
- `connectivity × 100` → 左侧"全局连通率" + 顶栏"拓扑连通"芯片
- `connectivity < 0.7` 时 → 触发全局红色闪屏告警特效

### 2.4 `positions.csv` — 高精度航迹文件
```csv
time,uav_id,pos_x,pos_y,rtk_drift_error
0,0,300.0,80.0,0.0
0,1,275.5,115.0,0.0
...
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `time` | int | 仿真时间戳 |
| `uav_id` | int | 无人机编号 (0~14) |
| `pos_x` | float | X 坐标（米），范围建议 0~600 |
| `pos_y` | float | Y 坐标（米），范围建议 0~600 |
| `rtk_drift_error` | float | RTK 定位漂移误差（米），>0 时前端渲染位置抖动 |

**前端消费方式**：
- `(pos_x, pos_y)` → 3D 沙盘中无人机的物理位置（前端做贝塞尔插值平滑）
- `rtk_drift_error > 0` → 坐标叠加随机偏移，模拟 RTK 失锁漂移

> **关键**：`positions.csv` 的行数 × 无人机数 = 总帧数。如有 10 个 time 值 × 15 个 uav_id = 150 行。前端进度条的总长度由 `distinct(time)` 的数量决定。

## 三、 前端组件功能清单

### 3.1 TopBar（顶栏）
- 左侧：旋转 SVG 菱形 Logo + "翼网全境 WING-NET OMNI" 品牌名
- 中间：三个实时状态芯片
  - **拓扑连通 XX%**：绿色，取自 `connectivity`
  - **同频冲突 N**：冲突时变红闪烁，取自前端实时判定的冲突对数
  - **引擎在线**：青色常亮
- 右侧：**SIM TICK** 仿真帧计数 + **SYSTEM** 真实系统时钟

### 3.2 LeftPanel（左侧面板）
- **QoS INDEX 健康圆环**：分数 = `avg_pdr × 100`，附带旋转粒子装饰
  - 颜色逻辑：≥90 绿 / ≥75 青 / ≥60 橙 / <60 红
- **核心指标面板**：
  - 平均 PDR → `avg_pdr × 100`%
  - P99 时延 → `avg_delay × 1.5` ms
  - 总吞吐 → `avg_throughput` Mbps
  - 活跃链路 → `num_links`
  - 全局连通率 → `connectivity × 100`%
- **活动节点档案**：15 架无人机列表，显示 UAV ID、信道标签（带对应颜色）、功率条、电池百分比

### 3.3 CenterSandbox（3D 沙盘 — 核心组件）
- **基于 Three.js WebGL** 的全 3D 可交互城市沙盘
- 功能清单：
  - 方块群城市建筑 + 赛博线框边缘 + 窗户灯光
  - 15 架无人机八面体模型 + 4 旋翼残影 + 发光点光源
  - 无人机按信道着色（CH1 青 / CH2 紫 / CH3 绿）
  - 机间通信连线：健康=青 / NLOS 遮挡=橙虚线 / 冲突=红色闪烁粗线
  - 数据流粒子动画沿连线流动
  - 飞行轨迹尾迹线（25 帧历史）
  - 干扰圈半透明红色柱体 + 线框
  - **Bloom 后处理辉光**效果
  - 鼠标交互：拖拽旋转 / Shift+拖拽平移 / 滚轮缩放
  - 点击无人机弹出详情

### 3.4 RightPanel（右侧面板）
- **PDR 折线图**（ECharts）：蓝色，最近 60 帧的 `avg_pdr × 100` 历史
- **吞吐量折线图**（ECharts）：紫色，最近 60 帧的 `avg_throughput` 历史
- **频谱柱状图**：统计当前帧 3 个信道各有多少架无人机
- **灾难 QoS 定量卡片**：
  - 🛡️ 可用性 → `avg_pdr × 100`%
  - ⏱ P99 上限 → `p99_latency_ms` ms
  - ⚡ 均方功率 → 所有无人机 `_pwr` 的均值

### 3.5 PlaybackBar（底部播放控制栏）
- 播放/暂停按钮（播放时带呼吸发光效果）
- 可拖拽进度条 + 发光跟踪圆点
- TICK 帧计数显示：`当前帧 / 总帧数`
- 倍速选择：0.5x / 1x / 2x / 4x
- 基础步长：100ms/tick（1x 速度下每 100ms 推进一帧）

### 3.6 UAVDetail（无人机详情弹窗）
点击 3D 沙盘中的无人机触发，显示：
- UAV 编号 + 信道标签 + 运行状态（正常/NLOS遮挡/同频冲突）
- 坐标 `(x, y)` + 发射功率 dBm
- 业务类型（高清图传 / 日常控制 / 数据采集，按 ID 轮转）
- 电池健康条（带充电条纹动画）
- 单机网络指标：PDR / 时延 / 吞吐量

### 3.7 ScenarioEditor（场景编辑器）
- 2D Canvas 编辑面板，支持两种模式：
  - 🏢 **建筑物模式**：鼠标拖拽画矩形，可调节高度（20~200m）
  - ⚡ **干扰圈模式**：从圆心拖拽画圆，可调节强度（10~100%）
- 实时 2.5D 等距预览（建筑带顶面和侧面）
- **"应用到 3D"按钮**：将编辑器中的建筑和干扰圈实时同步到 3D 沙盘
- **"导出"按钮**：输出 `scene.json` 文件（供后端接入 NS-3）
- **"清空/恢复默认"功能**

### 3.8 ResearchLab（科研实验室）
- 弹出式面板，ECharts 五维雷达图
- 数据源：`metrics_summary.csv`
- 对比 Graph Coloring / Greedy / Static 三种算法
- 五个维度：Throughput / Latency / Jitter / PDR / Topology Stability
- 底部"核心发现"结论文字

### 3.9 AlertOverlay（全局告警覆层）
- **自动触发条件**：当前帧 `conflicts > 0` 或 `connectivity < 0.7`
- 效果：
  - 全屏红色闪屏（0.3s 渐隐）
  - 屏幕边缘红色呼吸光晕
  - 全局画面抖动（0.2s）

## 四、 前端帧回放引擎核心机制

### 4.1 帧数据结构（TypeScript 类型）
```typescript
interface FrameData {
  tick: number                // 仿真全局时间戳
  QoS: {
    total_pdr: number         // 全网 PDR (0~1)
    p99_latency_ms: number    // P99 时延 (ms)
    throughput_mbps: number   // 全网吞吐 (Mbps)
    algo_compute_time_ms: number
  }
  uav_nodes: UAVNode[]        // 15 架无人机状态数组
  topology: {
    num_links: number         // 活跃链路数
    connectivity: number      // 全局连通率 (0~1)
  }
  conflicts: number           // 冲突对数
}

interface UAVNode {
  id: number                  // 0~14
  x: number                  // X 坐标 (米)
  y: number                  // Y 坐标 (米)
  channel: number            // 信道号 0/1/2
  is_conflict: boolean       // 是否正在同频冲突
  is_nlos: boolean           // 是否被建筑遮挡
  energy: number             // 电池 0~100
  is_active: boolean
  pdr?: number               // 单机 PDR
  delay?: number             // 单机时延 (ms)
  throughput?: number        // 单机吞吐 (Mbps)
  power?: number             // 发射功率 (dBm)
}
```

### 4.2 动态叠加判定（关键机制）
前端在读取每一"底座帧"后，会通过 `applyDynamicSceneToFrame()` 做**实时运行时补丁**：

1. **动态 NLOS 判定**：遍历场景编辑器当前放置的建筑物，计算无人机与建筑中心距离，若 `< (w+d)/2 + 15m` 则标记为 NLOS
2. **动态同频干扰判定**：双层遍历所有无人机对，若同信道且距离 `< 80m`，标记双方为 `is_conflict`
3. **QoS 重算**：冲突机 PDR 降至 ~0.65，NLOS 机 PDR 降至 ~0.82，健康机 PDR 保持 ~0.95

> 这意味着评委可以在路演时**实时拖拽建筑物**，画面中的无人机通信链路会立即响应变化。

## 五、 场景编辑器 JSON 导出格式

前端场景编辑器点击"导出"后生成的 `scene.json` 格式如下：

```json
{
  "buildings": [
    { "x": 180, "y": 150, "width": 50, "depth": 50, "height": 80 },
    { "x": 300, "y": 200, "width": 60, "depth": 40, "height": 100 }
  ],
  "interferenceZones": [
    { "x": 300, "y": 300, "radius": 80, "strength": 0.8 }
  ],
  "gridSize": 600
}
```

| 字段 | 单位 | 说明 |
|------|------|------|
| `buildings[].x, y` | 米 | 建筑左上角坐标 |
| `buildings[].width, depth` | 米 | 建筑宽度和进深 |
| `buildings[].height` | 米 | 建筑高度 |
| `interferenceZones[].x, y` | 米 | 干扰圈圆心坐标 |
| `interferenceZones[].radius` | 米 | 干扰半径 |
| `interferenceZones[].strength` | 0~1 | 干扰强度 |
| `gridSize` | 米 | 仿真区域尺寸（正方形，默认 600） |

**后端对接建议**：后端读取此 JSON，将建筑坐标转换为 NS-3 的 TopologyManager 障碍物配置，干扰圈转换为电磁干扰模型参数。

## 七、 后端需要确保的数据约束

| 约束项 | 说明 |
|--------|------|
| 无人机数量 | 前端硬编码为 **15 架**（ID 0~14） |
| 坐标范围 | `pos_x` 和 `pos_y` 建议在 **0~600 米** 范围内 |
| 时间戳一致性 | 4 个 CSV 文件的 `time` 列值必须**对齐**（相同的时间戳集合） |
| 信道值 | `uavX_ch` 取值为 **0、1、2**（不能超出，否则颜色映射失效） |
| CSV 头行 | 必须包含表头行，字段名严格匹配上述规范 |
| 文件路径 | CSV 文件放在 `public/data/` 目录下（Vite 静态资源） |
| 编码 | UTF-8，行尾 LF 或 CRLF 均可 |

## 九、 联调步骤

1. 后端将 4 个 CSV 文件复制到前端 `public/data/` 目录下
2. 启动前端：`npm run dev`
3. 浏览器打开 `http://localhost:5173?mode=csv`
4. 前端自动加载 CSV 并开始回放
5. 如果 CSV 加载失败（文件不存在/格式错误），前端自动回退到 Mock 模式
6. 后端可通过查看浏览器 Console 中的 `[WingNet] CSV 加载完毕, 共 XX 帧` 确认数据是否成功加载
